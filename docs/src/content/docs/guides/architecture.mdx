# Architecture

ryu_ldn_nx registers **three IPC services** simultaneously from `main.cpp`:

## Service Overview

| Service | Type | Role |
|---------|------|------|
| `ldn:u` | MITM | Intercepts Nintendo's LDN user service |
| `bsd:u` | MITM | Intercepts BSD socket calls for LDN subnet traffic |
| `ryu:cfg` | Custom | Tesla overlay config and status display |

All three share a **384 KB expanded heap** (`g_heap_memory`). `new`/`delete` route to this heap via `lmem::ExpHeap`. Do not grow buffers casually.

## Component Details

### ICommunicationService (`ldn:u` MITM)

The core LDN service. Games call its IPC commands (`CreateNetwork`, `Scan`, `Connect`, etc.) which are translated to RyuLdn protocol messages over TCP.

Key files:
- `sysmodule/source/ldn/ldn_icommunication.cpp` вҖ” main implementation (2100+ lines)
- `sysmodule/source/ldn/ldn_state_machine.cpp` вҖ” state machine (CommState transitions)
- `sysmodule/source/ldn/ldn_packet_dispatcher.cpp` вҖ” routes incoming packets to handlers
- `sysmodule/source/ldn/ldn_session_handler.cpp` вҖ” session lifecycle management
- `sysmodule/source/ldn/ldn_proxy_handler.cpp` вҖ” proxy data routing

### BsdMitmService (`bsd:u` MITM)

Intercepts BSD socket calls. Sockets targeting the LDN subnet **10.114.x.x** become proxy sockets whose traffic is tunneled through `ProxyData` packets. This is how PIA mesh protocol traffic (UDP broadcasts, unicast game data) reaches other peers without pcap.

Key files:
- `sysmodule/source/bsd/proxy_socket.cpp` вҖ” proxy socket implementation (bind, connect, sendto, recvfrom)
- `sysmodule/source/bsd/proxy_socket_manager.cpp` вҖ” socket registry, port allocation, data routing
- `sysmodule/source/bsd/ephemeral_port_pool.cpp` вҖ” dynamic port allocation (49152вҖ“65535)

### ConfigService (`ryu:cfg`)

Custom IPC service for the Tesla overlay. Exposes real-time LDN status, connection info, and live configuration changes.

Key files:
- `sysmodule/source/config/config_ipc_service.cpp` вҖ” IPC command handlers
- `sysmodule/source/ldn/ldn_shared_state.cpp` вҖ” shared state between MITM and config services

## PIA Protocol and Game Data

Games use Nintendo's PIA (Protocol Independent Application) library for mesh networking on top of LDN. After `CreateNetwork`/`Connect` establishes the LDN session:

1. Game calls `GetIpv4Address()` вҶ’ gets virtual LDN IP (e.g., `10.114.0.1`)
2. Game calls `GetNetworkInfo()` вҶ’ gets `NetworkInfo` with all nodes' IPs/MACs
3. Game identifies its own node by matching `GetIpv4Address()` result against `NetworkInfo.ldn.nodes[].ipv4Address`
4. Game opens UDP sockets via `bsd:u` вҶ’ MITM intercepts `bind`/`connect` to `10.114.x.x` вҶ’ creates `ProxySocket`
5. PIA sends broadcast UDP packets (mesh discovery) вҶ’ `RouteIncomingData()` delivers to **all** matching sockets
6. PIA sends unicast UDP/TCP packets вҶ’ `ProxySocket` вҶ’ `ProxyData` header вҶ’ TCP tunnel вҶ’ server вҶ’ peer

### Broadcast Delivery

PIA mesh discovery relies on UDP broadcasts reaching **every** listening socket on the same port. `RouteIncomingData()` uses `FindAllSocketsByDestination()` to fan out broadcast packets to all matching proxy sockets. The previous single-delivery `FindSocketByDestination()` broke PIA's mesh protocol in games like Smash Bros.

### IP Address Convention

All LDN IPs use **Ryujinx format** вҖ” big-endian read as `uint32` (e.g., `10.114.0.1` = `0x0A720001`). BSD `sockaddr_in.sin_addr` uses network byte order. Never double-convert.

## Packet Signaling

`HandleServerPacket` signals `m_response_event` **only** for actual response packets that `WaitForResponse` expects:

| Signals event | Packet types |
|---|---|
| Yes | `Connected`, `ScanReply`, `ScanReplyEnd`, `RejectReply`, `ProxyConnectReply`, `NetworkError` |
| No | `ProxyConfig`, `ProxyData`, `ProxyConnect`, `ExternalProxy`, `SyncNetwork`, `Ping` |

This prevents spurious wake-ups when `ProxyConfig` arrives before `Connected`.

## Connection Resilience

- **Fast first retry**: 200ms delay on first failure, then exponential backoff (1s initial, 2x multiplier, 30s cap)
- **Jitter**: Вұ10% randomization to prevent thundering herd
- **TCP keepalive**: 30s idle / 10s interval / 5 probes (Switch only)
- **Graceful disconnect**: `shutdown(SHUT_WR)` before `close()`
- **Auto-reconnect**: Unlimited retries via `ConnectionStateMachine`

```

  в”Ӯ                            в”Ӯ                           в”Ӯ
```

State transitions are signaled cross-thread via `os::Event` (AutoClear for response/scan, ManualClear for error).

## Memory Budget

- **Heap**: 384 KB expanded heap
- **Malloc buffer**: 1 MB for TLS heap central
- **Thread stacks**: 32 KB (MITM), 8 KB (config), 4 KB (log)
- **BSD sessions**: 14 concurrency limit
- **Total sysmodule budget**: ~10 MB across all Switch sysmodules
